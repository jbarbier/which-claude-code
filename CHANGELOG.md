# Changelog

## 0.2.0 — recursive self-invocation fix (security / reliability)

**If you are on 0.1.x, upgrade immediately.**

### The bug

The `UserPromptSubmit` hook shelled out to `claude -p --model haiku` to
generate a session title. The child `claude -p` inherited the user's
`~/.claude/settings.json` and enabled plugins, so its own `UserPromptSubmit`
event fired — re-invoking this hook, which spawned another `claude -p`, ad
infinitum. Detached and `disown`ed, the chain kept running after the user
closed Claude Code.

Observed in the wild:

- thousands of orphan `claude -p --model haiku` processes persisting after
  the UI was quit
- `~/.claude/projects/` growing to >12 GB, with thousands of JSONL shard
  files containing the title-generation instruction duplicated thousands of
  times
- token quota burned across all projects (the plugin is global)
- killing processes did not help — they instantly respawned

The only reliable emergency mitigation was moving the plugin cache dir aside
and then `pkill`-ing every matching process.

### The fix

`bin/update-session-title.sh` is rewritten to layer **six independent
recursion defenses**. Any single one of them is sufficient to stop the loop;
they are belt-and-suspenders so no future regression can re-enable the bug:

- **G1** Env-var recursion flag: internal `claude -p` calls carry
  `WHICH_CLAUDE_CODE_INTERNAL=1`; the hook exits immediately when it sees it.
- **G2** Content marker: every outbound payload carries a stable
  `[which-claude-code:title-generation-v1]` marker. If the inbound prompt
  contains it (or the literal instruction text), exit.
- **G3** Atomic `mkdir`-based per-session lock with 60s stale-lock cleanup.
- **G4** Per-session rate limit (2s cooldown).
- **G5** Plugin-owned bounded history file (5 lines × 200 chars max).
  The raw Claude transcript is **never** read — the previous implementation
  mined `transcript_path`, which itself grew via the recursion.
- **G6** Safe failure: 20s timeout, no retries, hard 2 KB payload cap,
  stderr routed only to the log file.

The product behavior is preserved: the title still updates after every
prompt when it makes sense. When a guard trips (rate limit, recursion,
failure) the hook exits cleanly without retrying.

### Tests + docs

- `tests/run.sh` — 17 regression cases covering every guard, session-id
  path-traversal, malformed JSON, payload size cap, history boundedness,
  title evolution.
- `README.md` — new "Defensive note for maintainers" section documenting
  the recursion hazard, and a manual verification checklist.

### Post-upgrade cleanup

If you were affected by the 0.1.x bug, your `~/.claude/projects/` directory
may still be bloated. Inspect before deleting:

```
du -xhd 1 ~/.claude/projects | sort -h | tail -10
```

Any project dir with thousands of small JSONL files is a casualty of the
recursion — those shards are effectively junk generated by the bug, not
real session history.
